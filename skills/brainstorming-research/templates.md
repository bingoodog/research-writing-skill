# 论文结构模板

根据论文类型自动推荐章节结构, 用户只需确认或微调。

## 本科毕业论文

```
1. 摘要
2. 绪论
   - 研究背景与意义
   - 国内外研究现状
   - 研究内容与方法
   - 论文结构安排
3. 理论基础/文献综述
4. 研究设计与方法
5. 实验与分析/实证研究
6. 结论与展望
7. 参考文献
8. 致谢
```

**字数要求**: 8000-15000字

# 项目文档结构模板

根据项目文档类型自动推荐结构, 用户只需确认或微调。

## 软件需求规格说明书（SRS）

```
1. 文档说明
   - 文档目的
   - 文档范围
   - 术语与缩略语
   - 参考文档
2. 项目概述
   - 项目背景
   - 项目目标
   - 用户群体与使用场景
   - 约束条件
3. 功能需求
   - 用户角色
   - 功能列表（FR-001 起编号）
   - 用例/用户故事说明
4. 非功能需求
   - 性能需求（响应时间、并发量等）
   - 安全需求
   - 可用性与可靠性需求
   - 可维护性与扩展性需求
5. 系统约束
   - 技术栈约束
   - 合规与法规要求
6. 验收标准
7. 附录
```

**页数参考**: 15-40页

## 概要设计文档（HLD）

```
1. 文档说明
2. 系统概述
   - 系统目标
   - 架构概览
3. 总体架构
   - 架构图（分层/模块）
   - 技术选型与理由
4. 模块划分
   - 各模块职责
   - 模块间接口关系
5. 数据架构
   - 数据模型概览
   - 数据流说明
6. 部署架构
   - 部署拓扑图
   - 基础设施要求
7. 关键设计决策
   - 问题 / 备选方案 / 选择理由
8. 非功能设计
   - 性能设计
   - 安全设计
   - 可扩展性设计
9. 附录
```

**页数参考**: 10-30页

## 详细设计文档（LLD）

```
1. 文档说明
2. 模块说明（每模块一节）
   - 模块职责
   - 类/函数设计
   - 接口定义（入参、返回值、错误码）
   - 内部处理逻辑（伪代码或流程图）
3. 数据库设计
   - 表结构定义
   - 索引设计
   - 关键SQL或ORM说明
4. 接口设计
   - API 列表（路径、方法、参数、响应）
   - 认证与授权机制
5. 异常处理设计
6. 附录
```

**页数参考**: 20-60页（取决于模块数量）

## 项目可行性分析报告

```
1. 执行摘要（结论先行）
2. 项目背景
   - 现状分析
   - 问题与需求描述
3. 项目方案
   - 方案描述
   - 技术路线
4. 可行性分析
   - 技术可行性
   - 经济可行性（成本收益分析）
   - 运营可行性
   - 时间可行性
5. 风险分析
   - 风险识别（风险矩阵）
   - 应对策略
6. 资源需求
   - 人力、设备、资金
7. 实施建议
   - 推荐方案
   - 下一步行动
8. 附录
```

**页数参考**: 10-25页

## 项目结题/总结报告

```
1. 执行摘要
2. 项目基本信息
   - 项目目标回顾
   - 实施周期
   - 团队成员
3. 目标完成情况
   - 计划目标 vs 实际完成（逐条对比）
   - 主要交付物清单
4. 工作总结
   - 主要工作内容
   - 技术方案与实现
5. 问题与解决方案
   - 遇到的主要问题
   - 解决方案与效果
6. 经验教训
   - 可推广的做法
   - 需要改进的地方
7. 后续建议
8. 附件清单
```

**页数参考**: 10-20页

## 项目进展报告

```
1. 报告摘要（本期核心信息）
2. 本期完成工作
   - 任务完成情况（完成数/计划数）
   - 里程碑达成情况
   - 主要成果
3. 下期工作计划
   - 计划任务
   - 里程碑目标
4. 风险与问题
   - 当前风险（概率/影响/应对）
   - 阻塞问题与处理进展
5. 资源与费用
   - 资源使用情况
   - 预算执行情况（如适用）
6. 需要决策或支持的事项
```

**页数参考**: 3-8页（定期周报/月报）

## 测试报告

```
1. 测试概述
   - 测试范围
   - 测试目标
   - 测试环境
2. 测试执行情况
   - 测试用例统计（总数/通过/失败/阻塞）
   - 测试覆盖率
3. 缺陷统计
   - 缺陷总数（按严重程度分类）
   - 缺陷趋势
   - 遗留缺陷清单
4. 测试结论
   - 质量评估
   - 是否建议发布
5. 遗留问题与处理计划
6. 附录
   - 测试用例清单（可链接）
   - 缺陷清单（可链接）
```

**页数参考**: 5-15页

## 使用说明

1. 用户确认文档/论文类型后, 自动加载对应结构
2. 展示结构时, 询问用户是否需要调整
3. 常见调整:
   - 增加/删减章节
   - 合并/拆分某些章节
   - 调整章节名称以匹配公司/学校模板
4. 记录最终确认的结构到 plan/outline.md
